# 오픈코드(OpenCode) 수정 요청 사항

## 1. 엑셀 가져오기 (Excel Import) 그룹핑 로직 수정
**문제점**: 
- 현재 엑셀 가져오기 시, '그룹' 열이 비어있으면 그룹이 끊긴 것으로 인식하여 `currentGroupName`을 `null`로 초기화함.
- 이로 인해 Step 3 등에서 하나의 그룹으로 묶여야 할 태스크들이 분리되거나 UI가 깨지는 현상 발생.

**요청 사항**:
- 엑셀의 '그룹' 셀이 비어있으면 **이전 행의 그룹을 상속**받도록 로직 변경.
- 단, **Step(스텝)**이 바뀔 때는 그룹을 초기화하여 이전 스텝의 그룹이 넘어오지 않도록 처리.
- Step 2, 3, 4의 '라운드 태스크'(`t3-round-1-pm` 등)가 엑셀 가져오기 시 단순 텍스트 매칭 실패로 인해 'New Custom Task'로 중복 생성되지 않도록 ID 파싱 로직 강화.

## 2. 진행률 바 (Progress Bar) 계산 수정
**문제점**:
- 태스크를 삭제하거나 숨겨도(예: Expedition 2 숨김), 내부적으로 `calculateTotalTasks` 함수가 이를 전체 모수(분모)에 포함하고 있어 진행률이 부정확함.
- 모든 태스크를 완료하지 않았는데도 100%가 뜨거나, 비활성 태스크가 카운트됨.

**요청 사항**:
- `calculateTotalTasks` 함수 수정.
- `project.task_states?.meta?.hidden_template_tasks` (숨겨진 템플릿 태스크)를 전체 카운트에서 **제외**.
- `is_expedition2_hidden`이 `true`일 경우 Step 4 태스크들을 전체 카운트에서 **제외**.
- 활성화된(보이는) 태스크만 분모로 계산하여 정확한 진행률(%) 표시.

## 3. 기타 요구사항
- 모든 문서 및 주석, 에이전트 응답은 **한국어**로 작성.
- 코드는 `App.tsx` 위주로 수정.
